# Docs Library

Primary documentation lives in `docs/`. These files are a good first stop for architecture and deployment.

## Key docs
- `docs/deployment.md` - full deployment guide
- `docs/environment-management.md` - environment philosophy and quick start
- `docs/architecture.md` - architecture overview and service layout
- `docs/agents.md` - repo and agent guidelines
- `docs/tasks.md` - active tasks and checklists
- `docs/auth-fixes.md` - auth/JWT/OAuth fixes and testing notes

## Doc folders
- `docs/accessibility/` - accessibility audits and results
- `docs/integrations/` - third-party integration plans
